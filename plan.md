# YouTube Multi Extractor 실행 계획

## 1. 목표

- 앱 이름을 `YouTube Multi Extractor`로 변경
- 화면 메뉴와 주요 UI 문구를 한글로 통일
- `yt-dlp` + `ffmpeg` 기반의 종합 추출 앱으로 확장
- 승인 전에는 구현하지 않고, 본 문서는 실행 계획만 정의

## 2. 현재 상태 요약

- 현재 FastAPI 앱 기준으로 지원 기능은 `오디오 추출`과 `자막 추출` 일부
- 현재 UI/문구는 영어 중심이며 앱명도 `YouTube Audio Extractor`
- 현재 백엔드는 오디오 작업 전용 잡 스토어와 오디오/자막 전용 요청 모델 위주
- 현재 자막 결과물은 `.vtt` 중심이라, 요구사항의 `.srt` 출력 지원이 추가 필요
- 재생목록/채널/영상 다운로드/고음질 노래 MP3 전용 프리셋은 아직 없음

## 3. 요구사항 기준 최종 기능 범위

### A. 오디오 추출

- 전체 또는 특정 구간 추출
- 출력 형식 선택 지원: 최소 `mp3`, `wav`, `aac`
- 필요 시 기존 지원 형식(`m4a`, `opus`) 유지 여부 검토

### B. 노래 MP3 추출

- 최고 음질 오디오 소스 사용
- MP3 변환 고정
- 메타데이터(title, artist, album 등 가능한 범위) 반영
- 앨범아트 또는 썸네일 임베드

### C. 영상 추출

- 전체 또는 특정 구간 자르기
- 해상도 선택 지원: `360p`, `480p`, `720p`, `1080p`, `1440p`, `2160p(4K)`, `4320p(8K)`
- 가용 해상도가 없을 때는 가장 가까운 하위 화질 또는 사용자 안내 필요

### D. 자막 추출

- 언어 선택
- 전체 또는 특정 구간 필터링
- 최종 출력 형식 `.srt`
- 공식 자막 우선, 없으면 자동 생성 자막 사용 검토

### E. 재생목록/채널 전체 다운로드

- 재생목록/채널 URL 입력 가능
- 위 기능(A~D)을 항목 단위로 반복 적용 가능
- 전체 작업 진행률과 개별 항목 실패 내역 표시 필요

## 4. 구현 방향

### 4-1. 아키텍처 정리

- FastAPI 앱을 기준 구현 대상으로 삼음
- 기존 오디오 전용 구조를 `작업 유형 공통 엔진`으로 일반화
- 작업 유형별 서비스 모듈을 분리
  - `audio`
  - `song_mp3`
  - `video`
  - `subtitle`
  - `playlist/channel batch`

### 4-2. UI 방향

- 상단 앱명: `YouTube Multi Extractor`
- 메뉴/모드 선택 한글화
  - `오디오 추출`
  - `노래 MP3 추출`
  - `영상 추출`
  - `자막 추출`
  - `재생목록/채널 다운로드`
- 각 모드별 옵션 입력 영역을 조건부 표시
- 장시간 작업을 고려해 진행률, 현재 단계, 완료/실패 건수 표시 강화

### 4-3. 공통 처리 원칙

- URL 유형 식별: 단일 영상 / 재생목록 / 채널
- 시간 구간 검증 공통화
- yt-dlp 메타데이터 조회와 ffmpeg 후처리 분리
- 다운로드 결과 파일명 정규화 유지
- 브라우저 다운로드 및 저장 폴더 선택 동작은 가능하면 기존 방식 재사용

## 5. 단계별 실행 순서

### 1단계. 앱명/메뉴/문구 재설계

- 백엔드 앱 title, HTML title, 헤드라인을 `YouTube Multi Extractor`로 변경
- 기존 영어 메뉴/버튼/상태 문구를 한글로 교체
- 현재 단일 폼 UI를 다기능 모드 UI로 재구성할 화면 설계 확정

산출물:

- 한글 메뉴 구조
- 모드별 입력 항목 정의
- 사용자 흐름 초안

### 2단계. 요청 모델 및 작업 유형 공통화

- 현재 `ExtractRequest`, `SubtitleRequest`를 기능별 또는 공통 기반 모델로 재설계

- 오디오 전용 `ExtractionJobStore`를 범용 작업 스토어로 확장

산출물:

- 작업 유형별 요청/응답 스키마
- 범용 백그라운드 작업 처리 구조

### 3단계. 오디오 추출 기능 확장

- 전체/구간 오디오 추출 유지
- `aac` 추가 및 포맷 정책 정리
- ffmpeg 변환 옵션을 형식별로 명확히 정의
- 진행률 문구를 한글화하고 공통 잡 구조에 연결

산출물:

- 일반 오디오 추출 API
- 포맷별 변환 로직

### 4단계. 노래 MP3 추출 전용 모드 추가

- yt-dlp 메타데이터와 썸네일 정보를 활용해 태그/앨범아트 임베드
- 일반 오디오 추출과 분리된 `음악용 프리셋`으로 제공

산출물:

- 노래 MP3 전용 서비스
- 메타데이터/앨범아트 임베드 처리

### 5단계. 영상 추출 기능 추가

## 7. 핵심 리스크와 대응

- 재생목록/채널은 단일 영상보다 시간이 오래 걸리므로 잡 구조 일반화가 선행되어야 함
- 8K/고화질은 원본 가용 여부에 따라 실패 가능성이 있어 fallback 정책 필요
- 메타데이터/앨범아트는 영상마다 품질 차이가 있어 누락 허용 기준 필요
- 자막은 언어/자동생성 여부에 따라 결과 편차가 있으므로 사용자 안내가 필요
- 기존 작업 중인 변경사항이 있어 구현 시 사용자 변경을 덮어쓰지 않도록 주의 필요
